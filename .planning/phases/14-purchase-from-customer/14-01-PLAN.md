1. In `backend/app/schemas/transaction.py`, add two new Pydantic models:
   - `PurchaseItem(BaseModel)`: `product_id: uuid.UUID`, `name: str`, `quantity: int` (must be > 0 via Field(gt=0)), `unit_price: Decimal` (must be > 0 via Field(gt=0))
   - `PurchaseCreate(BaseModel)`: `customer_id: uuid.UUID`, `items: list[PurchaseItem]` (must be non-empty via Field(min_length=1)), `notes: str | None = None`

2. Create `backend/app/services/purchase_service.py` with class `PurchaseService`:
   - Constructor takes `customer_repo: CustomerRepository`, `transaction_repo: TransactionRepository`, `product_repo: ProductRepository`, `ledger_repo: LedgerRepository`
   - Method `async def create_purchase(self, body: PurchaseCreate, creator_id: uuid.UUID) -> Transaction`:
     a. Fetch customer via `self._customers.get_by_id(body.customer_id)` -- raise HorizonException(404) if not found
     b. Compute `total = sum(Decimal(str(item.quantity)) * item.unit_price for item in body.items)`
     c. Create Transaction with: `customer_id=body.customer_id`, `created_by=creator_id`, `type=TransactionType.Purchase`, `currency=Currency.ILS`, `amount=-total` (NEGATIVE -- credits customer per signed-amount convention), `data={"items": [item.model_dump(mode="json") for item in body.items]}`, `notes=body.notes`
     d. Update `customer.balance -= total` (reduces debt), then `await self._customers.update(customer)`
     e. Save transaction: `await self._transactions.create(txn)`
     f. For each item in body.items:
        - Fetch product via `self._products.get_by_id(item.product_id)` with `with_for_update()` to prevent race conditions (use `select(...).where(...).with_for_update()` pattern -- check if ProductRepository supports this, otherwise use `session.execute(select(Product).where(Product.id == item.product_id).with_for_update())`)
        - `old_qty = product.stock_qty or 0`
        - `old_price = product.purchase_price or Decimal("0")`
        - `product.stock_qty = old_qty + item.quantity`
        - If `old_qty + item.quantity > 0`: `product.purchase_price = (old_qty * old_price + Decimal(str(item.quantity)) * item.unit_price) / (old_qty + item.quantity)`
        - `await self._products.update(product)`
     g. Create CompanyLedger entry: `direction="outgoing"`, `payment_method="balance_adjustment"`, `amount=total` (positive in ledger), `rep_id=creator_id`, `customer_id=body.customer_id`, `source_transaction_id=txn.id`, `date=txn.created_at.date()`, `status="pending"`
     h. Return the transaction object

   Follow the exact patterns from OrderService and PaymentService for repository calls. Import HorizonException from `app.core.errors`. Import Transaction, TransactionType, Currency from `app.models.transaction`. Import CompanyLedger from `app.models.ledger` (check actual import path). Import Product from `app.models.product`.

   IMPORTANT: The `with_for_update()` is critical for race condition prevention when two purchases sync simultaneously. If ProductRepository doesn't have a `get_by_id_for_update` method, use the session directly via `self._products._session` or add the method to the repository.
