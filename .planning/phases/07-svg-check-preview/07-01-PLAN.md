<task type="auto">
  <name>Task 1: Install to-words and create amountToWords.ts utility</name>
  <files>frontend/package.json, frontend/bun.lock, frontend/src/lib/amountToWords.ts</files>
  <action>
1. Install the `to-words` package:
   ```bash
   cd frontend && bun add to-words
   ```

2. Create `frontend/src/lib/amountToWords.ts` with the following implementation:

   - Import `ToWords` from `to-words`
   - Create three lazy-initialized `ToWords` instances, one per currency:

     **ILS config:**
     - `localeCode: "en-US"`
     - `converterOptions.currency: true`
     - `converterOptions.ignoreDecimal: false`
     - `converterOptions.ignoreZeroCurrency: false`
     - `converterOptions.doNotAddOnly: true`
     - `currencyOptions: { name: "Israeli Shekel", plural: "Israeli Shekels", symbol: "₪", fractionalUnit: { name: "Agora", plural: "Agorot", symbol: "" } }`

     **USD config:**
     - Same base options as ILS
     - `currencyOptions: { name: "US Dollar", plural: "US Dollars", symbol: "$", fractionalUnit: { name: "Cent", plural: "Cents", symbol: "" } }`

     **JOD config:**
     - Same base options as ILS
     - `currencyOptions: { name: "Jordanian Dinar", plural: "Jordanian Dinars", symbol: "د.أ", fractionalUnit: { name: "Fils", plural: "Fils", symbol: "" } }`

   - Store in a `converters` map: `Record<"ILS" | "USD" | "JOD", ToWords>`

   - Export a single function:
     ```typescript
     export function convertAmountToWords(
       amount: number,
       currency: "ILS" | "USD" | "JOD"
     ): string
     ```
     - If `amount` is falsy, zero, or negative, return `""`
     - Otherwise call `converters[currency].convert(amount)` inside a try/catch
     - On error, return `""`

   - **IMPORTANT:** After creating the file, run `console.log(convertAmountToWords(1250.50, "ILS"))` via a quick bun script to verify the output format matches the user's requirement: "One Thousand Two Hundred Fifty Israeli Shekels And Fifty Agorot" (or similar). If `to-words` outputs "Only" at the end, confirm `doNotAddOnly: true` is working. Adjust flags as needed.

   - Do NOT add `"and 00/100"` format here — the `to-words` library handles fractional units natively. The check SVG (Plan 02) will format the final display string.

  <name>Task 2: Add MICR E13B web font and @font-face declaration</name>
  <files>frontend/public/fonts/MicrE13b.woff2, frontend/src/index.css</files>
  <action>
1. Create the `frontend/public/fonts/` directory.

2. Obtain a MICR E13B font file. Two strategies in priority order:

   **Strategy A (preferred):** Search for a free/OFL-licensed MICR E13B woff2 font online. Try downloading from a reputable source. If a `.ttf` or `.otf` is found but not `.woff2`, convert it:
   ```bash
   # If woff2_compress is available:
   woff2_compress MicrE13b.ttf
   # Or use an online converter
   ```
   Place the file at `frontend/public/fonts/MicrE13b.woff2`.

   **Strategy B (fallback):** If no trustworthy free MICR font can be obtained programmatically, create a minimal placeholder file and add a comment in the CSS noting the font needs manual replacement. The MICR strip will fall back to `'Courier New', monospace` which is acceptable for this milestone (the strip is decorative, not magnetically scannable).

3. Add `@font-face` declaration to `frontend/src/index.css` — insert it BEFORE the `@tailwind base;` line:

   ```css
   @font-face {
     font-family: "MICR";
     src: url("/fonts/MicrE13b.woff2") format("woff2");
     font-display: block;
     font-weight: normal;
     font-style: normal;
   }
   ```

   **Why `font-display: block`:** The MICR strip is decorative. Blocking until font loads is acceptable — it prevents fallback monospace from flashing before the MICR font loads. The check preview is not in the critical rendering path.

4. Verify:

   - `bun run build` succeeds
