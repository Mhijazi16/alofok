3. Create `frontend/src/lib/imageCache.ts`:
   - Use `idb-keyval` with `createStore('alofok-images', 'blobs')` for a dedicated image blob store (per research recommendation -- simpler than Cache API, avoids service worker requirement per Pitfall 6)
   - Export `cacheProductImages(imageUrls: string[]): Promise<void>`:
     - For each URL, check if already cached (get by URL key), skip if exists
     - Fetch each URL, convert response to blob, store with `set(url, blob, imageStore)`
     - Use `Promise.allSettled` so one failure doesn't block others
     - Log warnings for failed fetches
   - Export `getOfflineImageUrl(url: string): Promise<string | null>`:
     - Try to get blob from imageStore by URL key
     - If found, return `URL.createObjectURL(blob)`
     - If not found, return null
   - Export `getImageCacheSize(): Promise<number>`:
     - Use `keys(imageStore)` to get all keys, then sum blob sizes via `get` for each
     - Alternatively use `navigator.storage.estimate()` for simpler total estimate
   - Export `clearImageCache(): Promise<void>`:
     - Use `clear(imageStore)` from idb-keyval
   - NOTE on object URL memory: the `useCacheSync` hook (Task 2) will manage lifecycle. Individual components should call `getOfflineImageUrl` and revoke URLs on unmount.
  </action>

  <name>Task 2: Create useCacheSync hook and wire PersistQueryClientProvider in main.tsx</name>
  <files>frontend/src/hooks/useCacheSync.ts, frontend/src/main.tsx</files>
  <action>
1. Create `frontend/src/hooks/useCacheSync.ts`:
   - Import `useCallback`, `useEffect`, `useState`, `useRef` from react
   - Import `useQueryClient` from `@tanstack/react-query`
   - Import `cacheProductImages`, `getImageCacheSize` from `@/lib/imageCache`
   - Import `syncQueue` from `@/lib/syncQueue`

   - Define `SyncTimestamps` interface: `{ products?: string; customers?: string; orders?: string; statements?: string }`
   - Define `SyncItemStatus` type: `'idle' | 'syncing' | 'done' | 'error'`
   - Define `SyncState` interface: `{ timestamps: SyncTimestamps; statuses: Record<keyof SyncTimestamps, SyncItemStatus>; pendingCount: number; cacheSize: number; isSyncing: boolean }`

   - localStorage key: `'alofok-sync-timestamps'`
   - Helper functions: `getSyncTimestamps(): SyncTimestamps` (parse from localStorage), `setSyncTimestamp(key: keyof SyncTimestamps)` (update one key in localStorage)

   - Define `PERSIST_QUERY_KEYS` as exported const: `['products', 'my-customers', 'route-day', 'delivery-orders', 'statement', 'collections', 'my-orders-today']`

   - Export `useCacheSync()` hook returning `{ syncState: SyncState; syncAll: () => Promise<void>; syncTimestamps: SyncTimestamps }`:
     - State: `statuses` (Record of SyncItemStatus), `isSyncing` boolean, `pendingCount` number, `cacheSize` number, `timestamps` (SyncTimestamps)
     - `syncAll()` async function:
       - Set `isSyncing = true`
       - For each data type (products, customers, orders, statements):
         - Set status to 'syncing'
         - Call `queryClient.invalidateQueries({ queryKey: [key] })` for the relevant query keys
         - After invalidation, set timestamp via `setSyncTimestamp(key)`, set status to 'done'
         - On error: set status to 'error', log warning, continue with next type
       - For products specifically: after invalidation, get products from query cache (`queryClient.getQueryData(['products'])`), extract all image_urls, call `cacheProductImages(allUrls)`
       - Update `cacheSize` via `getImageCacheSize()` + `navigator.storage.estimate()`
       - Update `pendingCount` via `syncQueue.count()`
       - Set `isSyncing = false`
     - Auto-refresh on online event:
       - `useEffect` listening to `window.addEventListener('online', handleOnline)`
       - `handleOnline` calls `syncAll()`
     - Auto-refresh on mount (app open):
       - `useEffect` on mount: if `navigator.onLine`, call `syncAll()` (debounced -- skip if already syncing)
     - Periodic pendingCount refresh: `useEffect` with interval every 30s to update `pendingCount`
     - Load timestamps from localStorage on mount

2. Modify `frontend/src/main.tsx`:
   - Add import: `PersistQueryClientProvider` from `@tanstack/react-query-persist-client`
   - Add import: `createIDBPersister` from `@/lib/queryPersister`
   - Import `PERSIST_QUERY_KEYS` from `@/hooks/useCacheSync`
   - Remove `QueryClientProvider` import (replaced by `PersistQueryClientProvider`)
   - Update `queryClient` defaultOptions: add `gcTime: 1000 * 60 * 60 * 24` (24 hours) alongside existing `staleTime` and `retry`
   - Create persister: `const persister = createIDBPersister();`
   - Replace `<QueryClientProvider client={queryClient}>` with:
     ```tsx
     <PersistQueryClientProvider
       client={queryClient}
       persistOptions={{
         persister,
         maxAge: 1000 * 60 * 60 * 24, // 24 hours
         dehydrateOptions: {
           shouldDehydrateQuery: (query) => {
             if (query.state.status !== 'success') return false;
             const key = query.queryKey[0];
             return typeof key === 'string' && PERSIST_QUERY_KEYS.includes(key);
           },
         },
       }}
     >
     ```
   - Keep everything else unchanged (Provider, App, StrictMode)

   IMPORTANT: The `PERSIST_QUERY_KEYS` array is exported from `useCacheSync.ts` as a plain const (not inside the hook) so main.tsx can import it without using the hook. This is the single source of truth for which queries get persisted.

   ANTI-PATTERNS TO AVOID:
   - Do NOT set `gcTime: Infinity` -- use 24h (per research Pitfall 1 and anti-pattern warning)
   - Do NOT persist mutation state -- only `status === 'success'` queries
   - Do NOT try to share the `alofok_offline` IndexedDB database with idb-keyval (per research Pitfall 3)
