<objective>
Add backend expense CRUD endpoints to the existing ledger router so Sales reps and Admins can create, list, and delete expenses stored as company_ledger entries.

Purpose: Provide the API layer for expense tracking. Expenses are outgoing ledger entries with payment_method=cash, enabling them to automatically appear in the existing daily cash report. Admin confirm/flag already works via the existing PATCH /ledger/status endpoint.

Output: Three new endpoints on the /ledger router: POST /expenses, GET /my-expenses, DELETE /expenses/{id}. Plus ExpenseCreateIn schema and supporting service/repo methods.
</objective>

     - Commit and return True if rowcount > 0, False otherwise
     - Use `update(CompanyLedger).where(CompanyLedger.id == entry_id, CompanyLedger.is_deleted.is_(False)).values(is_deleted=True)`

<task type="auto">
  <name>Task 2: Add expense service methods and API endpoints</name>
  <files>backend/app/services/ledger_service.py, backend/app/api/endpoints/ledger.py</files>
  <action>
1. In `backend/app/services/ledger_service.py`, add three methods to `LedgerService`:

   - `create_expense(self, rep_id: uuid.UUID, amount: Decimal, category: str, expense_date: date, notes: str | None = None, is_admin: bool = False) -> CompanyLedger`:
     - Create a CompanyLedger with direction="outgoing", payment_method="cash", amount=abs(amount), rep_id=rep_id, category=category, date=expense_date, notes=notes
     - Set status="confirmed" if is_admin else "pending"
     - Use `self._repo.create(entry)` to persist
     - Do NOT use `self.create_entry()` -- build the CompanyLedger object directly so we can set the conditional status

   - `get_rep_expenses(self, rep_id: uuid.UUID, expense_date: date) -> list[LedgerEntryOut]`:
     - Call `self._repo.get_by_rep_and_direction(rep_id, "outgoing", expense_date)`
     - Map each CompanyLedger to LedgerEntryOut (set rep_name=None, customer_name=None -- the frontend already handles missing names)
     - Return the list

   - `delete_expense(self, expense_id: uuid.UUID, caller_id: uuid.UUID) -> None`:
     - First fetch the entry: `entries = await self._repo.get_by_ids([expense_id])`
     - If not found or entry[0].rep_id != caller_id: raise HorizonException(404, "Expense not found")
     - If entry[0].status != "pending": raise HorizonException(400, "Only pending expenses can be deleted")
     - If entry[0].direction != "outgoing" or entry[0].source_transaction_id is not None: raise HorizonException(400, "Not a manual expense")
     - Call `self._repo.soft_delete(expense_id)`
     - Import HorizonException from app.core.errors

2. In `backend/app/api/endpoints/ledger.py`, add three endpoints:

   - `POST /expenses` (dependencies=[require_sales]):
     - Accepts `ExpenseCreateIn` body and `CurrentUser`
     - Determine is_admin: `current_user.get("role") == "Admin"`
     - REP_CATEGORIES = {"Food", "Fuel", "Gifts", "CarWash", "Other"}
     - If not is_admin and body.category not in REP_CATEGORIES: raise HorizonException(400, "Invalid category for Sales role")
     - Call `service.create_expense(rep_id=uuid.UUID(current_user["sub"]), amount=body.amount, category=body.category, expense_date=body.date, notes=body.notes, is_admin=is_admin)`
     - Return `{"id": str(entry.id)}`

   - `GET /my-expenses` (dependencies=[require_sales]):
     - Query param: `expense_date: date = Query(None, alias="date")`
     - If expense_date is None, use `date.today()`
     - Call `service.get_rep_expenses(rep_id=uuid.UUID(current_user["sub"]), expense_date=expense_date)`
     - Return the list directly

   - `DELETE /expenses/{expense_id}` (dependencies=[require_sales]):
     - Path param: `expense_id: uuid.UUID`
     - Call `service.delete_expense(expense_id, uuid.UUID(current_user["sub"]))`
     - Return `{"deleted": True}`

   - Add necessary imports: `uuid`, `ExpenseCreateIn` from schemas, `HorizonException` from app.core.errors
