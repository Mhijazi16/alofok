1. Add a new `CheckData` model ABOVE `TransactionOut`:
```python
class CheckData(BaseModel):
    bank: str | None = None            # existing: free-text bank name
    bank_number: str | None = None     # CHK-01
    branch_number: str | None = None   # CHK-02
    account_number: str | None = None  # CHK-03
    holder_name: str | None = None     # CHK-04
    due_date: str | None = None        # existing
    image_url: str | None = None       # existing
```

ALL fields MUST be `| None = None` — this is CRITICAL for backward compatibility (CHK-06). Old check rows with only `{bank, due_date}` must pass `CheckData.model_validate()` without errors.

2. Update `PaymentCreate.data` type from `dict | None = None` to `CheckData | None = None`.

3. Update `TransactionOut.data` type from `dict | None` to `CheckData | None`. Pydantic v2 with `from_attributes=True` will auto-validate the JSONB dict into CheckData on output. However, old rows with missing keys will have `None` for new fields — which is correct.

**In `backend/app/services/payment_service.py`:**

4. Update the check validation block (currently line 34-35). Replace:
```python
if body.type in _CHECK_TYPES and not body.data:
    raise HorizonException(400, "Check payments require data (bank, due_date)")
```
With:
```python
if body.type in _CHECK_TYPES:
    if not body.data:
        raise HorizonException(400, "Check payments require check data")
    if not body.data.bank_number:
        raise HorizonException(400, "bank_number is required for check payments")
    if not body.data.branch_number:
        raise HorizonException(400, "branch_number is required for check payments")
    if not body.data.account_number:
        raise HorizonException(400, "account_number is required for check payments")
```

5. Update the Transaction constructor call (currently line 48). Replace:
```python
data=body.data,
```
With:
```python
data=body.data.model_dump(exclude_none=True) if body.data else None,
```
This is CRITICAL because `body.data` is now a Pydantic `CheckData` model, not a plain dict. SQLAlchemy JSONB expects a dict, not a Pydantic model instance. `exclude_none=True` keeps the stored JSON clean (no null padding for optional fields that weren't provided).

1. Create a new Alembic migration (use `alembic revision -m "backfill_check_data_fields"` from `/backend`, or create manually). This is a DATA migration, not a schema migration — no `op.add_column` / `op.drop_column`.

The `upgrade()` function:
```python
def upgrade():
    op.execute("""
        UPDATE transactions
        SET data = jsonb_build_object(
            'bank',           data->>'bank',
            'bank_number',    data->>'bank_number',
            'branch_number',  data->>'branch_number',
            'account_number', data->>'account_number',
            'holder_name',    data->>'holder_name',
            'due_date',       data->>'due_date',
            'image_url',      data->>'image_url'
        )
        WHERE type = 'Payment_Check'
          AND is_deleted = false
          AND data IS NOT NULL
    """)
```

The `downgrade()` function:
```python
def downgrade():
    op.execute("""
        UPDATE transactions
        SET data = data - 'bank_number' - 'branch_number' - 'account_number' - 'holder_name'
        WHERE type = 'Payment_Check'
    """)
```

This reconstructs each check row's JSONB with all 7 keys present (null for new ones). Existing values are preserved. The downgrade strips only the new keys.
