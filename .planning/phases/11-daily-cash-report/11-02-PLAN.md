  <action>
Create `frontend/src/components/Admin/DailyCashReportView.tsx` with the following structure:

**Props:** `{ onBack: () => void }`

**State:**
- `reportDate: Date` (default: new Date())
- `expandedReps: Set<string>` (tracks which rep cards are expanded)
- `handedOverInputs: Record<string, string>` (per-rep handed-over amount input values)
- `flagInputs: Record<string, string>` (per-rep flag notes input values)
- `flaggingRep: string | null` (which rep's flag notes textarea is visible)
- `editingReps: Set<string>` (reps whose confirmed cards are in edit/undo mode)

**Data fetching:**
- `useQuery` with queryKey `["daily-cash-report", toLocalDateStr(reportDate)]` calling `adminApi.getDailyCashReport(toLocalDateStr(reportDate))`
- `useMutation` for confirmHandover — onSuccess: invalidate "daily-cash-report" queries, show success toast
- `useMutation` for flagHandover — onSuccess: invalidate "daily-cash-report" queries, show warning toast

**useEffect:** When `data` (report) loads, initialize `handedOverInputs` for each rep: if `rep.confirmation?.handed_over_amount` exists use that, otherwise use `rep.computed_net`. Also clear `editingReps` set.

**Date navigation:**
- `isToday = toLocalDateStr(reportDate) === toLocalDateStr(new Date())`
- `goToPrev`: setReportDate using `subDays` from date-fns
- `goToNext`: setReportDate using `addDays` from date-fns (disabled when isToday)
- Date display: use `format(reportDate, "PPP")` from date-fns
- Clicking the date text opens the existing DatePicker calendar with `disabled={{ after: new Date() }}`

**Discrepancy calculation (CASH-05):**
For each rep card, compute:
```
const computedNet = rep.computed_net;
const handedOver = parseFloat(handedOverInputs[rep.rep_id]) || 0;
const diff = Math.abs(handedOver - computedNet);
const pct = computedNet !== 0 ? diff / Math.abs(computedNet) : 0;
const hasDiscrepancy = pct > 0.05;
```

**Layout structure:**

1. **Header row:** Back button (ChevronLeft/ChevronRight based on RTL) + page title "Daily Cash Report"

2. **Date navigation bar:** `<Button variant="ghost" onClick={goToPrev}><ChevronLeft /></Button>` + date text (clickable to open DatePicker) + `<Button variant="ghost" onClick={goToNext} disabled={isToday}><ChevronRight /></Button>`. Centered layout. DatePicker rendered as a popover/dialog when date text is clicked.

3. **Grand totals bar:** A glass Card at the top showing 4 values in a row: Total Cash, Total Checks, Total Expenses, Grand Net. Use the design system's muted-foreground for labels, foreground for values, tabular-nums for numbers.

4. **Rep cards:** For each `rep` in `report.reps`, render a glass Card:
   - **Card border:**
     - If `rep.confirmation?.is_flagged` → `border-destructive ring-1 ring-destructive`
     - Else if `rep.confirmation && !editingReps.has(rep.rep_id)` → `border-green-500 ring-1 ring-green-500`
     - Else → default (no extra border)
   - **Card header:** Rep name + Badge showing status (Pending/Confirmed/Flagged) + expand/collapse chevron
   - **Card totals row:** 4 inline stats: Cash ₪X, Checks ₪X, Expenses ₪X, Net ₪X
   - **Multi-currency note:** If the backend returns totals that include non-ILS currencies, show them as-is (the backend sums all currencies together — for v1.2 this is acceptable, document as known limitation)
   - **Confirmation section (visible when NOT confirmed OR when in editingReps):**
     - Input field labeled "Handed Over Amount" pre-filled from handedOverInputs
     - If hasDiscrepancy: show warning text with percentage in warning color: `t("cash.discrepancy", { pct: (pct * 100).toFixed(1) })`
     - Two buttons: "Confirm Handover" (primary) and "Flag Discrepancy" (destructive outline)
     - Confirm button calls confirmMutation with { rep_id, report_date: toLocalDateStr(reportDate), handed_over_amount: parseFloat(handedOverInputs[rep.rep_id]) }
     - Flag button sets `flaggingRep` to show the notes textarea
   - **Flag notes section (visible when flaggingRep === rep.rep_id):**
     - Textarea for flag_notes
     - Submit Flag button (disabled until flagInputs[rep.rep_id]?.trim().length > 0)
     - Cancel button to clear flaggingRep
   - **Confirmed display (visible when confirmed AND NOT in editingReps):**
     - Show confirmed_at timestamp, confirmer_name
     - Show handed_over_amount
     - "Edit" button (ghost variant) that adds rep to editingReps set
   - **Flagged display:** Same as confirmed but also shows flag_notes text and flag icon
   - **Expandable section:** When chevron clicked, toggle rep in expandedReps set. When expanded, show a simple list of that day's payments and expenses. Since the main endpoint may not include individual items, show "X payments, Y expenses" counts from payment_count/expense_count. (Full transaction details can be added in a future iteration — for now, counts are sufficient.)

**Loading state:** Show Skeleton cards (3-4 skeleton cards).

**Empty state:** If report.reps is empty, show EmptyState with "No activity" message.

**Imports needed:**
- React: useState, useEffect
- react-i18next: useTranslation
- @tanstack/react-query: useQuery, useMutation, useQueryClient
- lucide-react: ChevronLeft, ChevronRight, Flag, CheckCircle2, AlertTriangle, Pencil

- UI components: Card, CardContent, Button, Badge, Input, Separator, Skeleton, EmptyState, PageContainer
