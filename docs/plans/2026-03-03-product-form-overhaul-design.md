# Product Form Overhaul Design

## Summary

Redesign the product creation/editing flow with: auto-generated SKUs, flexible discount (% or fixed), purchase price, autocomplete categories/trademarks, customizable option groups with price modifiers, and multi-image upload.

## Data Model Changes

### Product table modifications

- **Remove from user input**: `sku` — auto-generated by backend
- **Remove**: `discount_percentage`, `discounted_price`, `color_options`, `brand`
- **Add**: `trademark` (String, nullable) — replaces brand, autocomplete from existing
- **Add**: `purchase_price` (Numeric 12,2, nullable)
- **Add**: `discount_type` (String, nullable) — `"percent"` or `"fixed"`
- **Add**: `discount_value` (Numeric 12,2, nullable)
- **Change**: `image_url` (String) → `image_urls` (JSONB, list of strings, max 5, first=cover)
- **Keep**: `category` as plain string with autocomplete

### New table: product_options

| Column | Type | Description |
|--------|------|-------------|
| id | UUID (PK) | |
| product_id | UUID (FK → products) | |
| name | String | Option group name (e.g. "Color", "Size") |
| values | JSONB | `[{ label: string, price_modifier: number }]` |
| sort_order | Integer | Display ordering |
| created_at | DateTime | From BaseMixin |
| updated_at | DateTime | From BaseMixin |
| is_deleted | Boolean | From BaseMixin |

### Order line item changes

- Add `selected_options` JSONB to order items: `[{ name: "Color", value: "Red", price_modifier: 0 }]`
- Unit price = base_price + sum(modifiers), then discount if applicable

## SKU Auto-Generation

- Backend generates on create: `{CATEGORY_PREFIX}-{SEQ:03d}`
- Category prefix: uppercase, strip special chars, truncate to 5 chars. Fallback: `"ALF"`
- SKU is permanent — does not change if category changes later
- Duplicate products get fresh SKU (not "-copy" suffix)
- Frontend: SKU removed from form, shown read-only in detail views

## Pricing & Discount

- Price (required), Purchase Price (optional)
- Discount: segmented toggle (% / Fixed) + single value input
- Final price computed: percent → `price * (1 - value/100)`, fixed → `price - value`
- `is_discounted` toggle controls visibility to customers

## Categories & Trademark

- Both use combobox UI: search existing values or type new
- Backend endpoint: `GET /products/distinct-values?field=category|trademark`
- Returns `string[]` of unique non-null values from non-deleted products

## Product Options

- Dynamic list of option groups in the form
- Each group: name + list of values (label + price_modifier)
- Saved as `product_options` rows, replaced on update
- Order flow: dropdown per option group when adding to cart

## Multiple Images

- Up to 5 images per product, first is cover
- Grid of thumbnails with remove buttons, reorderable
- Upload endpoint unchanged (one file → one URL)
- Migration converts existing `image_url` to `[image_url]` array
- All display code reads `image_urls?.[0]` for cover
